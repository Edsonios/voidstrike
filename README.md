# VOIDSTRIKE — Warhammer 40,000 10e Simulator

A single-page, two-player hot-seat battle simulator for Warhammer 40,000 (10th edition rules approximation). Muster forces, set up terrain, deploy, and fight a five-phase battle round with interactive dice. Loads every faction and unit from Wahapedia's public data export.

> Unofficial fan tool. Warhammer 40,000 is © Games Workshop. Game data is © Wahapedia; this project is unaffiliated with both.

## Running it

### Option A — Deploy to Netlify (recommended: one-click data loading works)

The data-fetch "Fetch (auto)" button only works reliably when the site is served by Netlify, because it uses the bundled server function to pull Wahapedia data without CORS problems.

**Drag-and-drop:**
1. Go to https://app.netlify.com/drop
2. Drag the whole `voidstrike` **folder** onto the page (not just index.html).
3. Wait for deploy; open the URL. Done.

**Git (for ongoing edits):**
1. Push this folder to a GitHub repo.
2. In Netlify: "Add new site" → "Import an existing project" → pick the repo.
3. Build command: none. Publish directory: `.` (already set in `netlify.toml`).

### Option B — Open locally (no install)

Double-click `index.html`. Everything works **except** the auto-fetch button (no server). Use the **Upload CSV files** tab instead — see below.

## Loading all factions & units

Faction data **loads automatically** for every visitor from a committed snapshot file (`factions-data.json`) in your repo root. Visitors never fetch from Wahapedia and can't touch the fetch controls — they just open the page and all factions are there.

### Admin / maintainer workflow (password-gated)

The fetch, import and export controls live in the **Admin · Data Management** panel on the muster screen, locked behind a password.

- **Default password:** `voidstrike-admin`
- **To change it:** in `app.js` find the line `const ADMIN_HASH=...` and replace the number with the hash of your chosen password. Compute the hash by opening the browser console on your site and running:
  ```js
  (s=>{let h=0;for(let i=0;i<s.length;i++)h=(h*31+s.charCodeAt(i))|0;return h>>>0;})('your-new-password')
  ```
  Paste the resulting number as the new `ADMIN_HASH`, commit, done.

**To refresh the game data** (e.g. after a GW points/dataslate update):
1. Open the Admin panel, enter the password.
2. **Fetch & Load** (or **Re-fetch (update)** to bypass caches). Run it a couple of times if you want to be sure the data is complete and correct.
3. Click **Export Data Bundle** — this downloads `factions-data.json`.
4. Commit that file to your **repo root** (same folder as `index.html`). Netlify redeploys, and every visitor now auto-loads the new data.

> **Security note:** the password is a front-end gate — it keeps casual visitors out of the data controls, but it is *not* strong security (someone technical could bypass it via the page source or console). Don't reuse a sensitive password. For this use case (stopping randoms from triggering fetches) it's perfectly adequate.

> The data files used for a refresh: `Factions`, `Datasheets`, `Datasheets_models`, `Datasheets_wargear` (required), plus `Datasheets_models_cost` (points), `Datasheets_keywords` (keywords), `Datasheets_abilities` + `Abilities` (abilities).

### Data load order
1. `factions-data.json` committed in the repo (the snapshot you exported) — used for all visitors.
2. Browser localStorage (a returning visitor's cached copy) — instant.
3. Built-in Ultramarines + Chaos Space Marines only (if no snapshot exists yet).

## Game modes

- **Boarding Actions** (500 pts) — compact fight through a voidship interior with preset walls and hatchways.
- **Open War** (1000 pts) — larger clash on open ground; place your own terrain.
- **Sandbox** (no points limit, custom board size) — choose board dimensions (12–40 × 10–34 cells; 1 cell = 2″) and field any forces you like.

## Flow of a battle

1. **Muster** — pick factions per player, add units (or "Quick Muster").
2. **Terrain Setup** — choose a tool (Wall / Hatchway / Objective / Erase) and click the board. Walls block movement and line of sight; open hatchways let you move and fight through them. Press **Begin Deployment**.
3. **Deployment** — players alternate placing units in their highlighted Entry Zone.
4. **Battle rounds** — each player turn runs five phases:
   - **Command** — auto: +1 CP and Battle-shock tests.
   - **Movement** — click a unit, then a destination. Inside the green ring = Normal move; beyond it you **roll a D6 Advance**.
   - **Shooting** — click a unit to open fire control: pick a weapon (its range highlights on the board), pick a target, **roll the attacks** (hits → wounds → saves resolve in one batch). Fire each weapon separately.
   - **Charge** — click a unit to see legal targets within 12″, pick one, **roll 2D6**.
   - **Fight** — click any unit in Engagement Range, pick its target, **roll melee**. Chargers fight first.

Toggle **show dice rolls** in the right panel to see every die. **Auto-resolve Phase** plays the current phase for you (handy for watching or speeding through).

## Notes & limitations

- Weapon **ability tags** are parsed from Wahapedia (Rapid Fire, Sustained Hits, Lethal Hits, Devastating Wounds, Twin-linked, Torrent, Assault, Pistol, Hazardous, Anti-X) and affect the dice math. Unit **abilities** (Core/Faction/Datasheet) are imported and shown on the unit detail panel, but their rules effects are descriptive only — they aren't auto-applied.
- Weapons import at their **base profile**; wargear-option swaps aren't modelled. Squad sizes are inferred from the datasheet's unit composition — adjust with ± in the muster screen if needed.
- Stats, points, keywords and weapon tags come straight from Wahapedia; the rest is a faithful-but-simplified take on the core rules.

## Detachments, doctrines, stratagems & enhancements

A faction-agnostic detachment engine lives in `detachments.js`. Each faction is one entry; the file currently includes the full **Space Marines / Adeptus Astartes** set (17 detachments — Gladius, Anvil Siege, Ironstorm, Firestorm, Stormlance, Vanguard Spearhead, 1st Company, Librarius Conclave, Bastion, Orbital Assault, Ceramite Sentinels, Armoured Speartip, Headhunter, plus Ultramarines' Blade of Ultramar & Reclamation Force).

How it works in play:
- **Muster**: pick a Detachment per army from the new dropdown. Chapter-specific detachments only show for that chapter.
- **Command phase**: name your **Oath of Moment** target, and (for detachments with doctrines) select a **Combat Doctrine** / Psychic Discipline. Effects apply army-wide until your next Command phase.
- **Stratagem panel** (battle, left column): shows your **CP** and the stratagems whose timing window is open right now. Click to spend CP; target-self stratagems prompt for a unit. Effects feed into the live dice resolution.
- **Enhancements**: assigned to characters in code (`unit.enh`); their buffs apply to the bearer's unit.

Effects that are mechanically wired into combat include: Oath re-rolls, doctrine eligibility (shoot/charge after Advance/Fall Back), +1 Hit/Wound, granted weapon abilities ([SUSTAINED HITS], [LETHAL HITS], [DEVASTATING WOUNDS], [LANCE], [IGNORES COVER], [ASSAULT], [PRECISION], [ANTI-X]), Armour-of-Contempt AP worsening, -1 to be hit/wounded, Feel No Pain, invulnerable saves, bonus attacks/strength/AP, objective-control bonuses and sticky objectives.

### Adding more factions
Paste a faction's detachment rules, enhancements and stratagems and they're encoded into `DETACHMENT_DATA` using the same schema (see the top of `detachments.js` for the effect-code vocabulary). Currently **Space Marines**, **Black Templars**, **Necrons**, **Blood Angels**, **Dark Angels**, **Space Wolves**, **Deathwatch**, **Grey Knights**, **Orks**, **Tyranids**, **T'au Empire**, **Aeldari**, **Genestealer Cults**, **World Eaters**, **Death Guard**, **Chaos Daemons**, **Thousand Sons**, **Chaos Knights**, **Chaos Space Marines**, **Adeptus Custodes**, **Adeptus Titanicus**, **Adeptus Mechanicus**, **Agents of the Imperium**, **Drukhari**, and **Leagues of Votann** are filled in — every other army (Chaos factions, Orks, Tyranids, Necrons, Aeldari, Drukhari, T'au, Astra Militarum, Custodes, Sisters, Mechanicus, Knights, Agents, Votann, Genestealer Cults, and the divergent SM chapters' own detachments) still needs its data pasted in.

> Detachment rules text is GW's copyrighted material. The data file stores only the **mechanical effects** in brief, original wording — not the published flavour text.

Codex-compliant chapters (Ultramarines, Imperial Fists, Salamanders, Blood Ravens, etc.) already work out of the shared Space Marines bucket with full Oath.


### Strategic Reserves / Deep Strike subsystem (NEW — engine feature)
A real off-board reserves system now exists. A reserved unit is removed from the battlefield (it stays alive, so it is never counted as destroyed, but it is excluded from combat, targeting and scoring). At the start of a player's Movement phase the **Reinforcements step** prompts placement: click a cell more than 9″ from all enemy units to set a reserved unit down via Deep Strike, or Skip to hold it back. Units may also be sent into reserves mid-game by stratagems (new `toReserves` fx) or be granted Deep Strike (`grantDeepStrike` fx). The turn a unit arrives, its `setUpThisTurn` flag is set so "just deep-struck" bonuses (re-rolls, etc.) apply, then clears at its next Command phase. The AI/headless resolver auto-places reserves at a valid cell. This activates dozens of previously-dormant Deep-Strike / reserve-recall stratagems across every faction already encoded (Orbital Teleportarium, Guerrilla Tactics, Coordinated Strike, Relic Teleportarium, and more).

**Bug fix this round:** the movement-eligibility effects (`eligShootAfterAdvance`, `eligShootAfterFallBack`, `eligChargeAfterAdvance`, `eligChargeAfterFallBack`) were present in the data across many detachments (Stormlance, Bastion, Company of Hunters, Intractable, etc.) but were never wired into the shoot/charge gates — so those rules silently did nothing. A new `unitHasElig()` helper now resolves them (from army rule, detachment rule, active doctrine, active stratagems and enhancements, honouring conditions) and lifts the Advance/Fall-Back restrictions accordingly.

### Waaagh! subsystem (NEW — Orks army rule)
A once-per-battle army-wide buff toggle. A new **"Call the WAAAGH!"** button appears in the stratagem panel during an Ork player's Command phase. Calling it sets `waaaghActive` for that player until the start of their next Command phase, at which point it auto-expires (tracked by `waaaghTurnLeft`). While active, the Ork army rule's fx fire (gated on the new `waaaghActive` condition): charge after Advancing (via the eligibility system), +1 Strength and +1 Attack on melee weapons, and a 5+ invulnerable save. The panel shows a green "WAAAGH! ACTIVE" banner while it runs, and "already called" once spent. Many Ork detachment effects and stratagems gate on `waaaghActive` (Bully Boyz' Armed To Da Teef, More Dakka!'s shooting-phase Sustained Hits, etc.). New state: waaaghCalled / waaaghActive / waaaghTurnLeft per player (reset each battle). New condition: waaaghActive.

### Synapse subsystem (NEW — Tyranids army rule)
A *positional* army-wide buff (unlike the Waaagh! toggle). A Tyranids unit is "in Synapse Range" while it is a SYNAPSE model itself or within 6" of a friendly SYNAPSE model — computed live from board position by `unitInSynapse()`. While in Synapse Range: +1 Strength on melee attacks (army-rule fx gated on the new `withinSynapse` condition), and Battle-shock tests are rolled on 3D6 keeping the highest two. Many detachment effects and stratagems gate on `withinSynapse`. **Shadow in the Warp** is a once-per-battle panel button (Command phase) that forces every enemy unit to take a Battle-shock test, −1 within 6" of a Synapse model. The Synaptic Nexus detachment adds **Synaptic Imperatives**: each battle round you pick one of three (Augmentation 5+ invuln / Surging Vitality +1 Advance & Charge / Goaded to Slaughter +1 melee Hit) via panel buttons; each is usable once per battle and applies only to in-Synapse units (conditions `synAug`/`synSurge`/`synGoad`). The Feed-the-Swarm healing (Assimilation Swarm) runs in the Command phase via the reanimation hook: each HARVESTER heals a wounded friendly Tyranids unit within 6". New state: shadowUsed / synapticImp per player. New conditions: withinSynapse, synAug, synSurge, synGoad, targetInfantrySwarm, isHarvester, isTyranidWarrior.

#### Tyranids (added)

### Markerlight subsystem (NEW — T'au For the Greater Good)
A shooting-phase, two-role army mechanic. A new **"Mark targets (markerlights)"** button appears in the panel during a T'au player's Shooting phase. Pressing it makes every eligible unit an Observer that marks the closest visible enemy as **Spotted** (each enemy markable once); units with the MARKERLIGHT keyword additionally flag their target for cover-ignoring fire. Any T'au unit then shooting a Spotted enemy is **Guided** — it gets +1 to hit (the new `guided` condition), and against markerlight-Spotted targets the attack also ignores cover (the `guidedML` condition). Spotted marks reset at the start of each Shooting phase. The army-rule fx are gated on `guided`/`guidedML`; the AI/headless resolver auto-marks before shooting. New state: spotted / spottedML per player. New conditions: guided, guidedML, plus battle-round gates round3plus (Kauyon) and round1to3 (Mont'ka), and within9.

#### T'au Empire (added)
Six detachments: Kauyon (Patient Hunter — rounds 3-5 Sustained Hits, and Guided units ignore Hit modifiers vs Spotted targets), Mont'ka (Killing Blow — rounds 1-3 Assault, and Guided units get Lethal Hits), Retaliation Cadre (Bonded Heroes — Battlesuits gain +1 Strength within 12" and +1 AP within 9"), Kroot Hunting Pack (Hunter's Instincts plus a split 6+/5+ melee/ranged invuln), Auxiliary Cadre (Kroot/Vespid support auras, approximated as a flat ranged +1 AP), and Experimental Prototype Cadre (Superior Craftsmanship +6" range, with weapon-upgrade enhancements). The markerlight Guided loop, the battle-round-gated detachment rules, Bonded Heroes' distance bands, and the Kroot dual invuln are all live. Drone upgrades, the +6" range bonus, shaken/mortal-wound stratagems, and several reposition tricks are approximated or dormant pending those subsystems.


### Battle Focus subsystem (NEW — Aeldari army rule)
A per-battle-round **token economy** — the first resource-pool mechanic. At the start of each battle round an Asuryani army receives Battle Focus tokens scaled to battle size (2 / 4 / 6 for 500 / 1000 / 2000 pts), plus +1 for the Warhost detachment's Martial Grace and +1 per Timeless Strategist enhancement. The panel shows the current token count during the Aeldari player's turn with spend buttons for the Agile Manoeuvres: Swift as the Wind (+2", or +3" with Martial Grace), a generic reposition (D6+1"), Star Engines (the selected Vehicle's ranged weapons gain [ASSAULT]), Sudden Strike, and Flitting Shadows (no Overwatch). Spending decrements the pool and, for the movement manoeuvres, raises the unit's effective Move (now folded into the Movement-phase distance checks via a per-unit `_bfMove`). Star Engines sets a per-unit `_bfAssault` flag read by the combat-mod builder. Unspent tokens are wiped and the pool refreshed at the start of the next round (granted in both the post-deployment round-1 hook and the round-increment in endTurn). Per-unit Battle Focus flags clear at the start of each Movement phase.

#### Aeldari (added)
Twelve detachments — the largest roster so far: Warhost (Martial Grace, +1 token), Windrider Host (jetbike reserve recycling), Spirit Conclave (wraith constructs + Vengeful Dead, approximated), Guardian Battlehost (Defend at All Costs — +1 Hit near objectives), Ghosts of the Webway and Serpent's Brood (Harlequins — the latter granting Sustained Hits to Mounted/Vehicle weapons), Devoted of Ynnead (death-fuelled reactions), Seer Council (Strands of Fate; the Fate-dice economy isn't separately tracked but the Stratagems work), Aspect Host (Path of the Warrior as a re-roll-Hits-or-Wounds doctrine pair), Armoured Warhost (Skilled Crews — Vehicle ranged Assault), and the two Corsair detachments (Eldritch Raiders, Corsair Coterie) whose unit-applied Corsair Enhancements are modelled as regular enhancements. New conditions: objectiveContested, isHarleMountedVeh, strLowerThanT. The token economy, the move-allowance integration, Star Engines, Skilled Crews, the objective- and reserve-based detachment rules, and the various invulns are live; the many D6+1" reposition manoeuvres log intent and raise effective Move but (like all extra-move effects) don't physically reposition the AI token, and mortal-wound / Fate-dice / return-to-life effects remain approximated or dormant.


### Cult Ambush subsystem (NEW — Genestealer Cults army rule)
A resource-pool-fed resurrection-into-reserves mechanic that reuses the existing Strategic Reserves / Deep Strike machinery. At battle start a GSC army receives Resurgence points by battle size (6/10/14 for 500/1000/2000 pts, +2 per Deeds That Speak to the Masses enhancement). When an eligible GENESTEALER CULTS unit (non-Character, non-Vehicle) is destroyed, a death-hook in resolveAttacks automatically spends the appropriate number of Resurgence points (scaled to the unit's model count by `resurgenceCost`) to recreate an identical full-strength unit and place it into Cult Ambush — modelled as Strategic Reserves (`inReserve`, off-board) so it arrives in a later Movement phase through the same Deep Strike flow the Grey Knights and others use. Arrival bonuses (Host of Ascension's Sustained + Ignores Cover, A Chink in Their Armour's Lethal Hits) fire via the existing `setUpThisTurn` flag. The panel shows the live Resurgence pool during the GSC player's turn. New state: resurgence per player. New conditions: isMountedVehicle, within18, within6OfAlly.

#### Genestealer Cults (added)
Six detachments: Host of Ascension (A Perfect Ambush — reinforcements gain Sustained + Ignores Cover), Xenocreed Congregation (Unquestioning Fanaticism — Character-led units re-roll Advance/Charge), Biosanctic Broodsurge (Hypermorphic Fury — +1 Charge and a post-charge +1 Attack), Outlander Claw (Rapid Takeover — Mounted/Vehicle +1 OC), and the two allied-faction detachments Brood Brother Auxilia (Astra Militarum allies) and Final Day (Tyranids allies). The Brood Brother / Final Day army-composition rules (including allied units and the Astra Militarum overlapping-fire and Tyranid Catalyst auras) are not modelled — those auras are approximated as +1 Hit via the Oath-target mechanism or a within-6-of-ally check — but all their Stratagems work. The Resurgence economy, the death→reserves→arrival chain, the arrival bonuses, and the combat-shaping detachment rules are live; the literal Cult Ambush marker geometry, mortal-wound stratagems, and per-model healing remain approximated or dormant.


### Blessings of Khorne subsystem (NEW — World Eaters army rule)
A dice-pool gambling mechanic — the seventh distinct army-mechanic shape. At the start of each battle round a World Eaters army can make a Blessings of Khorne roll: eight D6 (rollKhorneBlessings), then spend doubles/triples to activate up to two of the six Blessings for the round. ## PLANNED (roadmap step 2, second half): datasheet-ability → fx join layer

Context: the original 10-step request (step 2) was "Include Faction Abilities, Core Abilities, Model Abilities — fetchable from Wahapedia." The **fetch half is already built** — the importer pulls `Datasheets_abilities.csv` + `Abilities.csv` alongside models/weapons/keywords/cost/wargear, resolves Core/Faction abilities via the shared Abilities table by `ability_id`, takes Datasheet/Wargear abilities inline, and shows them on the unit detail panel. The **unbuilt half is application**: that ability text is display-only; its rules effects are not auto-applied. (This is why, e.g., Drukhari Empowered is a blanket [SUSTAINED HITS 1] stand-in rather than each unit's own Pain ability — `TEMPLATES` carry no per-unit named abilities.)

Planned design (a join key, not a merge):
- Keep fetched ability *text* as the display/reference layer, keyed by `datasheet_id + ability name (normalised)`.
- Add a hand-authored **ability→fx map** keyed by that same stable key, each entry pointing at an `fx` list (the same vocabulary the detachment engine already uses).
- The map is a **sibling layer that must survive a re-fetch** (fetched data is overwritten on balance updates; the map is not). In the exported `factions-data.json` it lives as a separate top-level section the re-fetch preserves; in code it can equally be its own file. Join happens at load time; a missing/renamed ability degrades to "show text, apply nothing" rather than mis-applying.
- Consumers read the unit's mapped fx instead of generic abstractions. First intended consumer: Drukhari `empowerUnit()` reads the selected unit's actual Pain ability (display) and applies its mapped fx (mechanics), replacing the blanket Sustained Hits.

Enabler: the user can EXPORT `factions-data.json` from the live app (after a fetch) and UPLOAD it here, so the fx map can be authored against the real ability names/text/`parameter` values rather than guessed. Division of labour is unchanged: the engine + fx map are Claude's; the ability text lives in the fetched runtime data, never reproduced in Claude's output. This is a substantial, ongoing authoring effort (thousands of datasheet abilities) — build the join plumbing once, prove it on Drukhari Pain, then expand coverage faction by faction. Sequencing: finish the four remaining factions on the hand-fx track first, then build this layer as its own step.


### Yield Points & army-stance flip (NEW — Leagues of Votann army rule)
Prioritised Efficiency is a resource pool built on the Pain-token shape, with a distinctive threshold-driven **state flip**. ### Yield Points (NEW — Leagues of Votann army rule, threshold-flip resource pool)

### Acts of Faith — Miracle dice (NEW — Adepta Sororitas, substitute-a-die pool)
The thirteenth army-mechanic shape, and the first **dice-substitution** pool — distinct from the resource pools (spend a count) and the dice-pool gamble (read a roll). State: `miracleDice` is an array of pre-rolled *values* per player. You gain one each battle round (`grantMiracleRound`, fired at battle start and in `endTurn`) and one each time one of your own units is destroyed (`miracleOnDeath`, wired at the kill sites alongside the Pain/YP/EC death hooks); each gained die is rolled 1–6 and kept at that value. An Act of Faith substitutes a kept die into a roll. Threading substitution into the batch attack resolver (Hit/Wound/Damage/Advance/Charge/Save) isn't practical — the engine doesn't pause mid-roll — so the **honest, fully-wired** application is **Battle-shock**: the panel's Arm toggle (`sororToggleMiracleBshock`) sets a per-turn flag, and `doCommandPhase` substitutes the best Miracle die (`spendBestMiracle`) for one of the test's 2D6, removing it from the pool. The rest of the economy (round/death gains, valued dice, the pool display) is live; other Acts are resolved against the abstract dice engine and logged. Two detachments add round-pick layers reusing the doctrine/marker shapes: **Penitent Host** Vows of Atonement (`sororVow`, panel pick; Absolution in Battle flows via the `vowAbsolution` condition) and **Champions of Faith** Righteous units (`sororRighteous`, a Command-phase pick of up to 3 via the panel; buffs flow via the `selfRighteous` condition, lasting until the player's next Command phase). New helpers: `isSororitasArmy`, `gainMiracle`, `grantMiracleRound`, `miracleOnDeath`, `spendBestMiracle`, `sororToggleMiracleBshock`, `sororVowFor`, `chooseSororVow`, `isRighteous`, `toggleRighteous`. New conditions: `within6`, `selfRighteous`, `vowAbsolution`. The test confirmed round/death gains (and that non-Sororitas deaths grant nothing), best-die spend, the Battle-shock substitution consuming a die and passing the test, Blood of Martyrs' below-strength/half bonuses, Fervent Purgation's Assault + within-6 Strength, Righteous melee buffs, and the Absolution vow.
